# 架構一頁紙

五分鐘看懂：東西放哪、誰聽誰的。  
故事與好處見 [README](../../README.md)；指令見 [03](./03-Agent-指令範本.md)。

種子：**2026-07-06-v18**

---

## 1. 空資料夾最少要有什麼

**給 AI 的四樣：** `AGENTS.md` + `Bootstrap.md` + `MEMORY.md` + `skills/`

建議連同 `docs/for-reader/`、`README.md` 一起放，才找得到人話說明。

---

## 2. 建庫四步

```text
① 空目錄 + 本倉庫
② cd 庫根 → grok → 新對話
③ 貼 03 指令 ①（Bootstrap B0→B7）
④ 貼 03 指令 ② → 開第一個專題 →「請寫入」
```

步驟 ③ 只要求 `{庫根}/.grok/skills/` 有 kv-\*；**不要求**使用者目錄有。

---

## 3. 一張圖

```text
AGENTS.md ───── AI 每次先讀（去找誰）
    │
    ├── kv-* Skill ── 怎麼做（14 包，按需）
    │     母本 skills/   執行 .grok/skills/
    │
    ├── *_rules.md ── 完整條文（需要時才讀）
    └── Bootstrap.md ─ 空庫種子

MEMORY.md     僅 AI 交班
Project/      你的筆記與 MOC
Source/       原文
DailyChange   你的重大出貨紀錄
README + docs/for-reader/   給人看的說明
```

---

## 4. 記憶三分流

```text
對話 ──► MEMORY（AI）
     ──► DailyChange（你）
研究 ──► Project（正文）
```

| 白話 | 意思 |
|------|------|
| 請寫入 | 允許改檔；沒說＝只草稿 |
| 輕量審查 | 討論時過一輪，不寫檔 |
| 完整交付審查 | 重大寫入前較嚴的檢查 → 可記入 DailyChange |

---

## 5. 四層分工（Harness）

| 層 | 一句話 | 本庫 |
|----|--------|------|
| AGENTS | 去找誰 | `AGENTS.md` |
| Skill | 怎麼做 | `skills/` + `.grok/skills/` |
| 工具 | 有手腳 | 讀檔、終端、可選 MCP |
| Hook | 關鍵點約束 | 多靠 Skill 約定（請寫入前審查等） |

**優先序：** 你的指令 ＞ Skill／Canon ＞ AGENTS。

---

## 6. 五條鐵律

1. 未說「請寫入」→ 只草稿  
2. 規範只寫在規則層，不塞進研究筆記當第二套憲法  
3. 改完整規則 → 同步 Skill + Bootstrap 種子  
4. 改 `skills/` → 同步到 `.grok/skills/`（與開箱包）  
5. 打包公開前 → 確認 MEMORY／DailyChange 仍是種子  

---

## 7. 開箱包 vs 你建好後的庫

| | 剛 clone | 跑完建庫指令後 |
|--|----------|----------------|
| 規則能力 | 齊（在 Skill／種子裡） | 齊，且多了詳盡／MRC Canon 檔 |
| `.grok/skills/` | 通常還沒 | 應有 14 包 |
| 你的筆記 | 無 | 在 `Project/` 自長 |
