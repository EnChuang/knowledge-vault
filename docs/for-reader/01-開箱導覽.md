若還沒讀倉庫首頁，請先看 [README](../../README.md)（為什麼用、能得到什麼）。

種子：**2026-07-06-v18**

| 檔案 | 給誰 | 用途 |
|------|------|------|

## 你是哪一種安裝？

| 情境 | 手上有什麼 | 做什麼 |
|------|------------|--------|
| **A. 剛 clone 本倉庫** | AGENTS、Bootstrap、skills，可能還沒 `.grok/skills/` | 貼 [03 指令 ①](./03-Agent-指令範本.md) 建庫 |
| **B. 別人給你「已建好」的完整庫** | 已有 `.grok/skills/kv-*` | 在庫根開 `grok` 即可，不必再裝 |
| **C. 新電腦** | 已裝 Grok，資料夾同 A 或 B | 同 A／B；**不必**先把 kv-* 塞進使用者目錄 |

**關鍵**：終端工作目錄 = **庫根**（有 `AGENTS.md` 的那一層）。  
從別的資料夾開 grok，AI 往往找不到本庫 Skill。

## Skill 到底放哪？（最常誤解）

kv-\* **跟著知識庫資料夾走**，不跟著 Windows 帳號走。

| 路徑 | 要有 kv-\* 嗎？ |
|------|----------------|
| `{庫根}/skills/` | ✅ 母本（版控、給別人） |
| `{庫根}/.grok/skills/` | ✅ 執行時必須（建庫指令會複製） |
| `C:\Users\你\.grok\skills\` | ❌ **不必**（那是 Grok 內建 help、docx 等） |

```text
本倉庫
  skills/kv-*/          ← 母本
       ↓ 建庫時複製
  .grok/skills/kv-*/    ← Grok 實際讀這裡

C:\Users\你\.grok\skills\   ← 沒有 kv-* 完全正常
```

維護者若在**完整庫**改過 `skills/`：在庫根執行  
`python skills/kv-link-scan/scripts/sync_package.py`  
（見 [03 指令 ⑨](./03-Agent-指令範本.md)）。

## 記憶為什麼分三份？

```text
對話結束 ──► MEMORY.md     （給下一輪 AI，你可忽略）
         ──► DailyChange   （重大出貨時你看 Pass／改了什麼）
專題研究 ──► Project/      （長期正文，真正的知識）
```

| 檔 | 給誰 | 記什麼 |
|----|------|--------|
| `Project/` | 你 + AI | 專題怎麼寫 |
| `DailyChange.md` | **你為主** | 重大交付 |
| `MEMORY.md` | **僅 AI** | 交班符號 |

## 五分鐘建庫（推薦走 AI）

1. 倉庫放到空資料夾（該夾＝庫根）  
2. `cd` 到庫根 → `grok` → 新對話  
3. 貼 [03 指令 ①](./03-Agent-指令範本.md)  
4. 通過後貼指令 ② 開第一個專題；草稿 OK 再說「請寫入」

### 建庫後請勾

- [ ] `.grok/skills/` 下有 **14** 個 `kv-*/SKILL.md`  
- [ ] 庫根有四份 `*_rules.md` + `MEMORY.md` + `Project/_DailyChange/DailyChange.md`  
- [ ] 種子仍為 **2026-07-06-v18**（或你更新後的版本）  
- [ ] 可選：貼 [03 指令 ⑧](./03-Agent-指令範本.md) 讓 AI 自檢

## 若你要把這包再給別人

1. 在完整庫跑健檢（含開箱對齊）或 `sync_package.py`  
2. 確認 `MEMORY` 是空熱區、`DailyChange` 是空白範本  
3. 推 GitHub 或複製本倉庫  
4. 請對方：**先 README → 02 → 貼 03 指令 ①**；不要改對方的 `C:\Users\…\.grok\skills\`

## FAQ

**還需要 `*_rules.md` 嗎？**  
要。人讀、健檢、wikilink 靠它們；日常執行靠 Skill，不必每次讀完全文。

**新 Session 要手動指定 Skill 嗎？**  
通常不用。`AGENTS.md` 會對應任務該載哪些 kv-\*。

**完整庫改規則後，GitHub 上的包要更新嗎？**  
要。同步後更新 CHANGELOG，再 push／發 Release。

**MEMORY 我要讀嗎？**  
不必。看 `Project/` 與 `DailyChange` 即可。
