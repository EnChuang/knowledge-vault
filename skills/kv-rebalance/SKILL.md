# kv-rebalance — 專題重平衡

> **人類用語**：同專題又來新 Source／新重點時，**不要只往舊筆記尾巴貼**；先盤點覆蓋，再以**概念優先**重劃筆記邊界與 [[MOC]]，避免地圖被第一份素材帶偏。

前置：AGENTS 啟動前置。執行細節以本 Skill 為準。涉寫入 → **kv-method** L-全 + **kv-flow** §3/§4。更名／拆檔 → **kv-rename**。引讀新 Source → **kv-source**。正文格式 → **kv-project**。樹格式 → **kv-moc**。

## 1 觸發

| 觸發語／情境 | 行為 |
|--------------|------|
| 「重平衡」「重新分類」「地圖重排」「專題重整」 | 對指定專題跑完整 R0–R7（預設只出**計畫草稿**） |
| 同專題**新增 Source** 且用戶要整合進地圖 | 引讀後接本 Skill，不預設只 append 舊卡 |
| 發現一卡過肥、多卡重疊、樹被單一影片／一書章序綁死 | 建議觸發本 Skill |

**未說「請寫入」** → 只交覆蓋矩陣 + 重劃方案 + 遷移表（草稿）。  
**請寫入** → L-全 → 執行拆／併／更名／改 MOC／改正文。

## 2 核心原則

| 原則 | 說明 |
|------|------|
| **概念優先，非來源優先** | 樹節點＝知識子域（如「LDO 熱與效率」），不是「影片 A 筆記」「書 Ch.3」 |
| **地圖求均，不求鏡像 Source** | 新 Source 可補強舊節點；全新點才開新葉；禁止為每份 Source 各建平行樹 |
| **一葉一題、知識長文自足** | 每子筆記仍滿足 kv-project 80%＋敘事主導；過肥則拆，碎片則併 |
| **MOC 與筆記同次對齊** | 重劃必含：簡介、樹、文獻來源；待延伸不預填規劃 |
| **先計畫後動刀** | R4 方案須用戶認可範圍；再等「請寫入」 |

### 均衡判準（R3 自檢）

| # | 不均信號 | 目標方向 |
|---|----------|----------|
| B1 | 單卡承擔 ≥3 個可獨立面試／设计子域 | 拆葉 |
| B2 | 兩卡 >40% 重疊論點 | 併或重劃邊界 |
| B3 | 整棵樹深度／廣度被**單一 Source 目錄**決定 | 改為概念軸 |
| B4 | 新 Source 要點無處可掛，只能開「雜項」 | 調軸或新葉（語意化命名） |
| B5 | 文獻表很長但樹只有 1 葉 | 該拆未拆 |

## 3 流程 R0–R7

```
R0 鎖定專題
R1 盤點現況（MOC + 全子筆記 + 文獻表）
R2 新 Source 引讀 → 要點清單（kv-source 1–3；無法全文則標限制）
R3 覆蓋矩陣 + 均衡診斷（B1–B5）
R4 重劃方案（新樹、拆併更名、遷移表）——草稿對齊用戶
R5 用戶確認範圍
R6 請寫入 → L-全 → 執行（rename/write/moc/link）
R7 §4 簡報 + 可選 MEMORY
```

### R0 鎖定專題

- 專題資料夾與 `{專題} — MOC.md` 必須可解析  
- 跨專題素材只入「延伸／文獻」，不強行並樹（除非用戶明確要求合併專題）

### R1 盤點現況

讀且只讀該專題（止搜）：

| 產出 | 內容 |
|------|------|
| 樹快照 | MOC 巢狀 wikilink 現況 |
| 葉清單 | 每檔：`title`、約略主題邊界、體量（過肥／適中／碎片） |
| 文獻表 | MOC「文獻來源」+ 筆記銜接中的 Source／外鏈 |

### R2 新 Source 要點

- 走 **kv-source** 引讀：能讀完則讀完；影片無字幕等 → **標明限制**，不發明口播  
- 產出**要點清單**（每點一行：概念名 + 一句 + 可選出處錨）  
- 大綱與用戶對齊後再進 R3（引讀步 3）

### R3 覆蓋矩陣

列 = 要點（舊卡已有 ∪ 新 Source）；欄 = 現有筆記（+「無歸屬」）。

| 標記 | 含義 |
|------|------|
| **已有** | 舊文已覆蓋，新 Source 僅可加例證／數字 |
| **補強** | 舊葉該補一段，不必新檔 |
| **全新** | 無葉可掛 → 新葉或擴界後再掛 |
| **衝突** | 舊說與新 Source 矛盾 → 標註，正文寫取捨或待查，禁止默默覆蓋 |

附 B1–B5 診斷結論（過肥／重疊／來源綁架／無掛點／文獻—樹失衡）。

### R4 重劃方案（草稿必含）

1. **目標概念軸**（3–9 個一層子域為宜；過深再二層）  
2. **目標 MOC 樹**（巢狀 wikilink；只寫計畫中的檔名，執行前可標「（計畫）」於聊天，**落庫樹禁預填不存在葉**——執行時先建檔再掛樹，或同次請寫入一併建立）  
3. **檔案動作表**

| 動作 | 舊 | 新 | 說明 |
|------|----|----|------|
| keep | … | … | 邊界微調 |
| split | 一檔 | 多檔 | 遷移段落對照 |
| merge | 多檔 | 一檔 | 保留名／消歧 |
| rename | 舊名 | 新名 | 走 kv-rename 清單 |
| create | — | 新名 | 新概念葉 |
| retire | 舊名 | — | 內容遷出後刪或改 redirect 說明 |

4. **文獻掛載**：來源進 MOC 文獻表；正文概念葉可多源，避免「一源一葉」  
5. **不做清單**：明確不寫入的枝（防 scope 膨脹）

### R5–R7 確認與執行

| 步 | 規則 |
|----|------|
| R5 | 用戶認可軸與動作表；有爭議先改方案 |
| R6 | 「請寫入」→ **kv-method** L-全 → **kv-flow** §3 → 順序建議：建新檔／寫遷移正文 → **kv-rename** 全庫對照 → **kv-moc** 更新四段 → **kv-link** → 自測（新／大改葉） |
| R7 | §4 簡報：樹前後對照、動作數、殘留風險；階段收工問 MEMORY |

## 4 與其他 Skill

| Skill | 分工 |
|-------|------|
| **kv-source** | 新 Source 讀完／大綱／不發明 |
| **kv-project** | 葉正文知識長文、命名、自測、卡點譬喻 |
| **kv-moc** | 四段、M1–M10；重劃後樹＝已存在筆記 |
| **kv-rename** | 更名／搬移全庫 grep |
| **kv-link** / **kv-link-scan** | 連結一致性；重劃後可掃一次 |
| **kv-flow** | 請寫入前後檢查 |
| **kv-method** | 每任務 L-輕；請寫入 L-全 |
| **kv-audit** | 大重劃後可選健檢，不取代本 Skill |

## 5 反模式

| 禁止 | 原因 |
|------|------|
| 新 Source → 只在最舊一卡文末 append 長文 | 地圖永久偏重第一源 |
| 每部影片／每章一張卡且永不交叉 | 來源鏡像，難複習 |
| 未請寫入就拆檔更名 | 違反 kv-flow |
| 重劃時預填 MOC「待建立」大樹 | 違反 kv-moc 不預填 |
| 發明影片口播／文獻未讀段落 | 違反證據原則 |

- …
